Rodríguez Rayo Ángel

Sobre mí
Egresado de la Maestría de Sistemas Computacionales con especialidad en Tecnologías WEB, siendo de tipo profesionalizante, en donde nuestro objetivo principal es el de solucionar uno de los problemas que pudiese tener una organización o empresa particular utilizando tecnologías web para su desarrollo.

Dentro de esta maestría aprendimos a desarrollar una aplicación de forma completa, comenzando desde el análisis de requerimientos y su clasificación (funcionales y no funcionales), diseño de diagramas de software, como pueden ser:

• Diagramas de modelado de procesos.
• Diagramas de casos de uso.
• Diagramas de entidad asociación.
• Diagramas de clases.
• Diagramas de secuencias.
• Diagramas de despliegue.

Conocimos las más importantes y recomendadas metodologías para el desarrollo de software, tanto agiles como tradicionales, principalmente la metodología Scrum.

Durante mi estancia de estudio desarrolle un proyecto de software para una empresa dedicada a proveer soluciones de comunicación, mi proyecto consta del desarrollo de una aplicación web SPA, utilizada para la gestión administrativa de uno de sus servicios, la tarea principal de esta aplicación es la de notificar automáticamente a los clientes que su periodo de pago esta por expirar. Mi aplicación web está desarrollada en el framework ASP .NET Core 2.1 para su back-end y el framework de Vuetify para su front-end. La Arquitectura de la solución consta de tres proyectos, para generar abstracción entre cada una de las capas de la aplicación, dividiéndolo en un proyecto entidades, datos y web, dentro del proyecto web utilizamos el patrón MVC.

En nuestro front-end utilizamos el framework Vuetify para el desarrollo de las vistas de la SPA, utilizando la librería de axios para acceder a los controladores creados.

Para la seguridad de nuestra aplicación implementamos la utilización de tokens con una duración de 30 minutos, de cumplirse este plazo sin actividad el token de eliminar, requiriendo que el usuario vuelva a iniciar sesión sin importar el rol que tenga, se implemento un gestor de roles y usuarios, donde si su estatus era inactivo, no podría iniciar sesión.
